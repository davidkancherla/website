How it works

i.
Parse the noise
Claude reads the messy notes — fragments, abbreviations, side-tangents — and extracts the actual signal: decisions, action items, owners, dates.
ii.
Structure the output
Returns a strict JSON shape: summary paragraph, action items array (with owner, due date, priority), decisions list, parking lot. No surprises in the schema.
iii.
Format for the destination
One click to copy as Slack-ready bullets, email-ready paragraphs, or paste straight into Linear / Asana / Notion. Same data, three formats.
